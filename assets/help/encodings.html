Supported Encodings

Currently, DroidEdit supports the following encodings: UTF-8, UTF-8 with BOM, Western (ISO-8859-1), Western (Windows-1252), Eastern (ISO-8859-2), Eastern (Windows-1250), Cyrillic (ISO-8859-5), Cyrillic (Windows-1251), Greek (ISO-8859-7), Greek (Windows-1253), Traditional Chinese (Big5), Korean (EUC-KR), Thai (Windows-874), Japanese (Shift JIS), Japanese (EUC-JP), UTF-16 and ASCII.

If you need support for other encodings, don't hesitate to ask.

Default Encoding

You can define the default encoding using the Editor → Default Encoding preference.

Opening and Creating Files

If you use different encodings all the time, you can set up DroidEdit so it asks for the correct file encoding every time you create or open a file by changing the Editor → Always Ask Encoding preference.

If you're using the old menu configuration you can also long press the New file menu icon to create a new file using a different encoding than the default one or long press the Open file menu icon for the same reason.
