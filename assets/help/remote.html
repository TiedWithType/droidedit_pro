Remote Files

How to connect and manage files in remote locations using Dropbox, Box, SFTP, FTP and FTPS.

Dropbox

Dropbox is a file hosting service that offers cloud storage and file synchronization. To use Dropbox with DroidEdit you first have to create a Dropbox account at www.dropbox.com.

To open and save files on your Dropbox account, you just have to choose the Dropbox option when executing one of those actions.

If you have the Dropbox app installed in your phone, it will be used to authorize DroidEdit to access your Dropbox account, if you don't have the app, the browser will be used for it. After authorizing DroidEdit, you don't have to do it again.

After the initial authorization process, you just use your Dropbox file system like you would normally use your local one. If you wan't to revoke DroidEdit access to tour account, just use the Remote Files → Disconnect from Dropbox option in preferences.

If you don't use Dropbox and don't want the Dropbox option to show every time you open or save a file, just use the Remote Files → Show Dropbox Option option in preferences.

SFTP

SFTP (SSH File Transfer Protocol) is a network protocol that provides file access, file transfer, and file management functionalities over a secure connection.

To use SFTP, you need a server with SSH capabilities and an account. You can add a connection to a SFTP server by using the preference Remote Files → Add Remote Server. You should set Server Type to SFTP, give it a Server Name, that will be used in the select file system menu, and enter the Server Address (either as an IP address or a hostname) and Username. If you don't set a Password, DroidEdit will ask for it every time you connect to the server.

The Server Port and Initial Folder are optional. You can also set a Base URL so DroidEdit knows how to preview remote scripts in the browser.

If you prefer to use a private key to authenticate against the remote server, you can set the Private Key Path. This path should point to a file containing the private key of a DSA or RSA key pair using the OpenSSH format. If the key is protected with a passphrase it will be asked every time you connect to the server. You can also set a global private key that will be used with all your SFTP connections using the Remote Files → Global Private Key Path preference.

If DroidEdit is asking for a passphrase even when you didn't set one, then you are probably using a private key in the wrong format.

After setting all needed parameters you can press Test to test your connection, save it using Save or cancel the whole process using Cancel. After saving the connection, it will appear in the choose file system menu every time you open or save a file.

To edit an SFTP connection, just go to Remote Files → Server and press on the desired server. You'll then be able to edit, test and delete your connection.

If you chose not to save passwords with your SFTP connections, they will be saved in memory only while DroidEdit is running. If you want DroidEdit to forget those temporary passwords just use the Remote Files → Forget Session Passwords preference.

FTPS and FTPES

FTPS and FTPES are extensions to the File Transfer Protocol (FTP) that add support for the Transport Layer Security (TLS) and the Secure Sockets Layer (SSL) cryptographic protocols.

You can create FTPS and FTPES connections the same way you create an FTP connection. You'll just have to check the Use FTPS and the Use FTPES options when creating the connection.
