Supported Languages

Currently, DroidEdit supports syntax highlighting for the following languages: Assembly X86, C, C++, C#, Clojure, CSS, Delphi, DTD, Fortran, Groovy, HTML, Java, JavaScript, JSON, JavaServer Pages, LaTeX, Lisp, Lua, Pascal, Perl, PHP, Python, Ruby, Scala, Shell Script, SQL and XML.

DroidEdit's syntax highlighting is based on the RSyntaxTextArea (fifesoft.com/rsyntaxtextarea) open source project under a modified BSD license. RSyntaxArea syntax files are created using JFlex, a lexical analyzer generator for Java, and, although extremely powerful, are not trivial to write.

If you need support for a different language, you can either try writing your own JFlex syntax file (and contribute for the open source project) or try asking the developers of RSyntaxArea to support your language scheme. Either way, you should start by reading this post in RSyntaxArea's forums.

Turning Off Syntax Highlighting

You can turn syntax highlighting off using the Editor → Syntax Highlighting preference.

Forcing Syntax Highlighting

DroidEdit uses the file extension to determine which syntax scheme to use. If you would like to use a particular syntax scheme with a file extension not recognized by DroidEdit, you should:

  1. Select File Actions → Force Syntax... from the menu.
  2. Select the desired language
  3. Select if you want to keep the association between the current file extension and the selected language permanent.
