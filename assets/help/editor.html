DroidEdit Help

Editing

The main function of DroidEdit is to serve as a simple text and code editor. In this page some of the editing functions are explained.

Moving Around

You can press on the screen to move your cursor to any desired position. You can also use swiping movements to scroll up/down and left/right.

If you have an external keyboard, you can use the arrow keys ← ↑ ↓ → to move you cursor around the currently opened file.

You can also go directly to the beginning of a line using a keyboard shortcut Ctrl+Q or to the end of the line Ctrl+E.

To go directly to a specific line, you can either use the Goto Line menu item or use a shortcut shortcut Ctrl+L.

You can jump to the next/previous word using Ctrl+→ and Ctrl+←.

Cut, Copy and Paste

To select text you can press and hold where you want to select text and then move the handles to fine tune the text selection. You can also use your keyboard Shift+→↑↓←.

You can also select the next/previous word using Ctrl+Shift+→ and Ctrl+Shift+←.

To cut, copy and paste text you have three choices. If you selected text using touch and hold, you should see a contextual action bar with those actions. If not, you can either use a menu item Edit → (Cut, Copy or Paste) or a shortcut Ctrl+X, C or V.

You can also duplicate or delete an entire line using the shortcuts Ctrl+D or Ctrl+Shift+D.

Undo / Redo

To undo the last modification to the current file, you can either press the undo icon or use the default shortcut Ctrl+Z.

To redo, you can either press the redo icon or use the default shortcut Ctrl+Y.

You can configure the number of undo levels in the settings Editor → Undo Level. You can use 0 for infinite undos, but be aware that this might make DroidEdit slower if you make a lot of changes.

Writer Mode

DroidEdit was developed primarily for code editing. Many users also use it for writing text in a variety of languages. When in writer mode, DroidEdit highlights typos and shows keyboard suggestions. This mode is not active by default as it would make code editing more cumbersome.

To turn writer mode on, you can either use the menu item Writer Mode or change it in the settings Editor → Writer Mode.

Read Only Mode

If you are using DroidEdit just to read code and the virtual keyboard is having a nasty habit of popping up when you are just trying to scroll, you can turn on the Read Only mode.

To turn it on, you can either use the menu item Read Only or change it in the settings Editor → Read Only Mode.

Tab Key Behavior

Some developers prefer to use tab characters to indent their code while others prefer spaces. To accommodate both kind of users, DroidEdit allows you to change the tab key behavior. By default, pressing the tab key inserts a tab character. To use spaces instead of tabs, use the preferences Editor → Use spaces as Tab and Editor → Number of spaces.

Take a look at the shortcuts section to learn how you can insert tabs if your keyboard (physical or virtual) lacks the tab key.

Line Breaks

You can change the default character (CR, LF or CR + LF) used for a line break when saving files using the preference Editor → Line Breaks.
